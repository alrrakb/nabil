## 📌 القسم الأول: فكرة المشروع وشرح الصفحات

### 💡 فكرة المشروع إيه؟

تخيل إنك بتشتغل في معهد كبير زي معهد دلتا العالي، وفيه أقسام كتير زي الإدارة، والأرشيف، وتكنولوجيا المعلومات، وغيرهم. كل يوم بيتبادل موظفين ومدراء مراسلات رسمية مع بعض — طلبات، تقارير، تعميمات، موافقات — وده كان بيحصل إما على ورق أو على إيميلات مش منظمة.

**المشروع ده بيحل المشكلة دي!**

هو نظام ويب متكامل بيخلي أي موظف يبعت مراسلة رسمية لزميله أو لقسم تاني، والمدير يقدر يوافق عليها أو يرفضها أو يأرشفها، وكل حاجة بتتسجل وبتتتبع بشكل منظم. 

**باختصار:** بدل الورق والإيميلات الفوضوية، عندك نظام واحد متكامل بيتحكم في كل المراسلات الداخلية والخارجية للمعهد.

### 📄 شرح صفحات النظام

---

#### 1. 🔐 صفحة تسجيل الدخول (`/login`)

دي أول صفحة هتشوفها لما تفتح النظام. فيها خانة الإيميل وكلمة المرور.

**بتعمل إيه بالظبط؟**
- بتدخل إيميلك وباسووردك وبتضغط "دخول"
- النظام بيتحقق من بياناتك مع Supabase Auth
- لو البيانات صح، بيجيب بروفايلك من قاعدة البيانات (اسمك، دورك، قسمك)
- لو الباسوورد اتغير ومحتاج تغييره تاني، النظام بيعمل اللازم
- لو البيانات غلط، بيظهرلك رسالة خطأ واضحة

**مين بيستخدمها؟** كل أنواع المستخدمين (موظف، مشرف، مدير، سوبر أدمن)

---

#### 2. 🏠 صفحة الرئيسية / لوحة التحكم (`/`)

دي صفحة البداية بعد تسجيل الدخول. هتلاقيها ملخص شامل لحالة المراسلات.

**بتعمل إيه بالظبط؟**
- بتعرضلك **بطاقات إحصائية** في الأعلى زي:
  - إجمالي المراسلات
  - المراسلات المعلقة (Pending)
  - المراسلات الموافق عليها
  - المراسلات المؤرشفة
- **رسم بياني دائري** بيوضح توزيع المراسلات حسب الحالة
- **رسم بياني عمودي** بيوضح توزيع المراسلات حسب القسم
- **جدول المراسلات الأخيرة** بيعرض آخر المراسلات اللي اتعملت

**مين بيستخدمها؟** كل المستخدمين — بس كل واحد بيشوف البيانات الخاصة بيه أو بقسمه حسب صلاحيته

---

#### 3. 📥 صفحة صندوق الوارد (`/inbox`)

دي الصفحة الأساسية اللي بتشوف فيها كل المراسلات اللي جاتلك أو بعتها.

**بتعمل إيه بالظبط؟**
- بتعرضلك **جدول بكل المراسلات** فيه أعمدة:
  - الرقم المرجعي (رابط قابل للضغط للتفاصيل)
  - الموضوع
  - النوع (داخلي / خارجي)
  - المرسل
  - المستلم
  - الحالة (ملون حسب الحالة)
  - الأولوية (ملون حسب الأولوية)
  - التاريخ

- **فلترة وبحث:** تقدر تفلتر حسب الحالة، حسب النوع، وتبحث بالرقم المرجعي أو الموضوع

- **تصدير Excel:** زر بيصدر الجدول كله لملف Excel بضغطة واحدة

**مين بيستخدمها؟** كل المستخدمين

---

#### 4. ✏️ صفحة إضافة مراسلة جديدة (`/add`)

دي الصفحة اللي بتعمل منها مراسلة جديدة.

**بتعمل إيه بالظبط؟**
- **فورم شامل** بيطلب منك:
  - موضوع المراسلة
  - نوع المراسلة (داخلية / خارجية)
  - الأولوية (عادي / عاجل)
  - المستلم (موظف أو قسم)
  - نص المراسلة (اختياري)
  - **رفع ملف مرفق** (PDF، صورة، إلخ) — بيتخزن على Supabase Storage

- لما تضغط إرسال، النظام:
  - بيولد **رقم مرجعي تلقائي** بصيغة `CORR-YYYY-XXXX`
  - بيحفظ المراسلة في قاعدة البيانات
  - بيبعت **إشعار** للمستلم

**مين بيستخدمها؟** كل المستخدمين المسجلين

---

#### 5. 📑 صفحة تفاصيل المراسلة (`/correspondences/:id`)

دي الصفحة اللي بتشوف فيها تفاصيل مراسلة معينة لما بتضغط على رقمها المرجعي.

**بتعمل إيه بالظبط؟**
- بتعرضلك **كل تفاصيل المراسلة:** الموضوع، النص، المرسل، المستلم، التاريخ، الملفات المرفقة
- **سجل التاريخ (History):** بيوضح كل الأكشنز اللي اتعملت على المراسلة دي (تم الإنشاء، تم الموافقة، تم الرفض، إلخ)
- **أزرار الأكشن** (بتظهر حسب صلاحيتك):
  - ✅ **موافقة** — للمدير أو المشرف
  - ❌ **رفض** — للمدير أو المشرف
  - 📦 **أرشفة** — للمدير أو المشرف
  - 🗑️ **حذف** — للمدير فقط
- **تحميل الملف المرفق** لو فيه مرفق

**مين بيستخدمها؟** كل المستخدمين، بس الأكشنز بتختلف حسب الدور

---

#### 6. 📦 صفحة الأرشيف (`/archive`)

صفحة بتعرض كل المراسلات اللي اتأرشفت.

**بتعمل إيه بالظبط؟**
- بتعرض **جدول بالمراسلات المؤرشفة** مع تفاصيل الأرشفة (مين أرشفها، إمتى، وسبب الأرشفة)
- **بحث وفلترة** في الأرشيف
- كل صف قابل للضغط للدخول على تفاصيل المراسلة

**مين بيستخدمها؟** المدير والمشرف فقط

---

#### 7. 📊 صفحة التقارير (`/reports`)

صفحة تحليلية بتعرض إحصائيات ورسوم بيانية.

**بتعمل إيه بالظبط؟**
- **رسوم بيانية تفصيلية** توضح:
  - عدد المراسلات حسب كل قسم
  - عدد المراسلات حسب الحالة
  - توزيع الأنواع (داخلي / خارجي)
- **تصدير التقرير** كملف Excel أو PDF

**مين بيستخدمها؟** المدير والمشرف فقط

---

#### 8. 🏢 صفحة الأقسام (`/departments`)

صفحة إدارة الأقسام الموجودة في المعهد.

**بتعمل إيه بالظبط؟**
- **عرض كل الأقسام** في كروت ملونة مع اسم القسم وكوده
- **إضافة قسم جديد** (اسم + كود + لون مميز)
- **تعديل** بيانات قسم موجود
- **حذف** قسم (مع تحذير لو فيه موظفين أو مراسلات مربوطة بيه)

**مين بيستخدمها؟** المدير فقط (`admin`)

---

#### 9. 👥 صفحة الموظفين (`/employees`)

صفحة إدارة حسابات الموظفين في النظام.

**بتعمل إيه بالظبط؟**
- **عرض كل الموظفين** في جدول مع الاسم، الإيميل، الدور، القسم
- **إضافة موظف جديد:** بتحدد اسمه، إيميله، دوره، قسمه، وباسوورد مؤقت — النظام بيعمله حساب تلقائي في Supabase Auth
- **تعديل** بيانات موظف موجود
- **حذف** موظف
- **🔑 إعادة تعيين كلمة المرور** (يظهر فقط لـ Super Admin): بيولد باسوورد جديد تلقائي ويعرضه مرة واحدة

> **مهم:** حساب `master@delta.edu.eg` (السوبر أدمن) مش بيظهر في القائمة دي أبداً — ده عن قصد للحماية.

**مين بيستخدمها؟** المدير فقط (`admin`)

## ⚙️ القسم الثاني: التقنيات والأسلحة اللي استخدمناها

### 🗂️ هيكل المشروع (Monorepo)

المشروع مبني كـ **Monorepo** باستخدام `pnpm workspaces`. يعني عندنا أكتر من برنامج في ريبو واحد بيشاركوا كود مشترك:

```
nabil/
├── artifacts/
│   ├── correspondence/   ← الواجهة الأمامية (Frontend)
│   └── api-server/       ← الباك إند / السيرفر
├── lib/
│   ├── db/               ← تعريف قاعدة البيانات والجداول
│   ├── api-zod/          ← Schemas التحقق من البيانات
│   ├── api-spec/         ← مواصفات الـ API
│   └── api-client-react/ ← Hooks جاهزة للتعامل مع الـ API
```

**ليه Monorepo؟** عشان مش محتاجين نكتب نفس الكود مرتين — مثلاً تعريف نوع بيانات "المراسلة" بنكتبه في `lib/api-zod` مرة واحدة وبيستخدمه الـ Frontend والـ Backend مع بعض.

## 🚀 القسم الثالث: إزاي تشغل المشروع

#### الخطوة 9: اعمل إعداد Nginx

```bash
sudo nano /etc/nginx/sites-available/nabil
```

الـ Config:

```nginx
server {
    listen 80;
    server_name [IP-ADDRESS-OR-DOMAIN];

    # الواجهة الأمامية
    location / {
        proxy_pass http://localhost:9000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # الـ API
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_http_version 1.1;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Host $host;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/nabil /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 🔑 القسم الرابع: الدخول للنظام والحسابات

### إزاي تسجل الدخول؟

1. افتح المتصفح وادخل على لينك النظام
2. هتلاقي صفحة تسجيل الدخول
3. حط الإيميل وكلمة المرور
4. اضغط "دخول"
5. النظام هيوديك لصفحة الداشبورد تلقائياً

---

### 👑 حساب السوبر أدمن (Super Admin)

ده أقوى حساب في النظام — عنده كل الصلاحيات.

| البيان | القيمة |
|--------|--------|
| **الإيميل** | `master@delta.edu.eg` |
| **الدور** | Super Admin |
| **المميزات الخاصة** | مش بيظهر في قائمة الموظفين، بيشوف زر 🔑 إعادة تعيين كلمة المرور لأي موظف |

> **تنبيه:** الحساب ده لا يُشارك مع أي حد، هو لأغراض الإدارة الكاملة للنظام فقط.

---

### 👤 حسابات الموظفين

#### قاعدة الإيميل

```
[كود الموظف]@delta.edu.eg
```

**مثلاً:** لو كود الموظف هو `EMP001`، يبقى إيميله:
```
EMP001@delta.edu.eg
```

#### كلمة المرور الافتراضية

```
Password123!
```

> ⚠️ **مهم جداً:** أول ما أي موظف يدخل بالباسوورد الافتراضي، لازم يغيره فوراً.

---

### 🔄 إعادة تعيين كلمة مرور موظف (Super Admin فقط)

1. دخل بحساب السوبر أدمن `master@delta.edu.eg`
2. روح صفحة **الموظفين** (`/employees`)
3. دور على الموظف اللي عايز تعمله Reset
4. اضغط على أيقونة المفتاح 🔑 الصفراء جنب اسمه
5. هيظهرلك ديالوج تأكيد
6. بعد التأكيد، النظام هيولد باسوورد جديد تلقائي ويعرضه مرة **واحدة بس**
7. **اكتبه فوراً** وبعته للموظف، لأنه مش هيظهر تاني!

## 📝 ملاحظات ختامية

- **الـ Codebase** كله مكتوب بـ TypeScript — مش هتلاقي ملف `.js` خالص في الـ Source Code
- **الـ API** بيستخدم نظام **Zod** للتحقق من البيانات في كل endpoint — يعني مش ممكن بيانات غلط توصل لقاعدة البيانات
- **الإشعارات** بتتبعت تلقائياً لما حد يبعت مراسلة أو يعمل أكشن عليها
- **الملفات المرفقة** بتتخزن على **Supabase Storage** مش على السيرفر مباشرة
- **الأمان:** كل الـ Endpoints محمية، والـ Role-based access control بيتطبق في الـ Backend مش بس الـ Frontend
